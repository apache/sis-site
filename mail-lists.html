Mailing lists

We welcome you to join mailing lists and let us know about your thoughts or ideas about Apache SIS. Everyone can subscribe, but note that all emails sent to those lists are archived and publicly readable on the web. See Public Forum Archive Policy for more information.

  • Cross-project mailing lists
    • Geospatial mailing list
  • Apache SIS mailing lists
    • User mailing list
    • Developer mailing list
    • Issues mailing list
    • Commit mailing list

Cross-project mailing lists

Geospatial mailing list

This is where peoples can ask questions or discuss about geospatial topics that are not specific to Apache SIS. Those topics can include questions about international standards, participation to upcoming geospatial events or collaboration with communities outside Apache.

  • Subscribe: geospatial-subscribe@apache.org
  • Post (after subscription): geospatial@apache.org
  • Unsubscribe: geospatial-unsubscribe@apache.org
  • Archives: Pony mail

Apache SIS mailing lists

User mailing list

This is where users can ask questions about how to use SIS.

  • Subscribe: user-subscribe@sis.apache.org
  • Post (after subscription): user@sis.apache.org
  • Unsubscribe: user-unsubscribe@sis.apache.org
  • Archives: Pony mail

Developer mailing list

This is where the community hangs out and SIS development happens.

  • Subscribe: dev-subscribe@sis.apache.org
  • Post (after subscription): dev@sis.apache.org
  • Unsubscribe: dev-unsubscribe@sis.apache.org
  • Archives: Pony mail

Issues mailing list

This is where changes on JIRA issues are reported.

  • Subscribe: issues-subscribe@sis.apache.org
  • Unsubscribe: issues-unsubscribe@sis.apache.org
  • Archives: Pony mail

Commit mailing list

This is where the community receives automated notifications of any changes to the sis code and documentation.

  • Subscribe: commits-subscribe@sis.apache.org
  • Unsubscribe: commits-unsubscribe@sis.apache.org
  • Archives: Pony mail
